    API reference¶

    The complete public surface of the valgebra package. Every name is re-exported from the top-level valgebra namespace.

    Compiling and checking¶

    validator builtin ¶

    validator(schema: object) -> CompiledValidator

    Compile a schema into a reusable validator.

    The schema is any supported form: a type or typing annotation (int, list[str], int | None, Literal[...], a TypedDict, a dataclass, an Annotated refinement, ...), a native form (a [T] list, a {T} set, a {K: V} mapping, an all-string-key dict record, or any constant as a literal), or another compiled validator.

    Parameters:

    Name Type Description Default
    schema object

    The schema to compile.

    		 required

    Returns:

    Type Description
    CompiledValidator

    An immutable CompiledValidator for the schema.

    Raises:

    Type Description
    NotImplementedError

    If the schema uses an unsupported form (for example a recursive class, which must be written with lazy).

    CompiledValidator ¶

    A compiled, immutable schema validator.

    Build one with validator, or with a combinator such as union, intersect, or lazy. Then check values with validate, is_valid, or cast, and JSON documents with validate_json or is_valid_json.

    Validation is a membership test against the set the schema denotes: the value is never copied or coerced. A validator never changes after it is built and is safe to share across threads. Its repr is the annotation that produces it, and it can be copied with copy.copy/copy.deepcopy.

    cast method descriptor ¶

    cast(obj: object) -> object

    Validate obj and return it unchanged.

    The explicit conversion entry point. Because validation is a membership check rather than a coercion, the returned object is exactly the input; cast exists so converting code reads distinctly from checking code.

    equivalent method descriptor ¶

    equivalent(other: object) -> bool

    Whether this schema and other denote the same set — mutual inclusion.

    other is any schema spec or compiled validator. Sound, like is_subtype: True only when the two are provably equivalent, whatever their syntax (bool | int is equivalent to int).

    Parameters:

    Name Type Description Default
    other object

    The schema to compare, as a spec or validator.

    		 required

    Returns:

    Type Description
    bool

    True if the two schemas are equivalent, else False.

    is_empty method descriptor ¶

    is_empty() -> bool

    Whether the schema is unsatisfiable — provably empty, so is_valid returns False for every value.

    Decided soundly: True only when no value can belong to the schema — an unsatisfiable intersection, a fixed sequence with an impossible position, a record with an impossible required field, or a recursive schema with no base case (a mandatory self-reference that can never bottom out). It never reports a satisfiable schema as empty; for forms it cannot decide it returns False.

    Returns:

    Type Description
    bool

    True if the schema denotes the empty set, else False.

    is_subtype method descriptor ¶

    is_subtype(other: object) -> bool

    Whether every value of this schema is also a value of other — set inclusion, the subtyping relation.

    other is any schema spec or compiled validator. The decision is sound: True only when the inclusion provably holds (bool is a subtype of int, list[bool] of list[int]); for forms it cannot decide — Or sequences, recursive references, class checks across schemas — it returns False rather than a relation it cannot justify.

    Parameters:

    Name Type Description Default
    other object

    The candidate supertype, as a schema spec or validator.

    		 required

    Returns:

    Type Description
    bool

    True if this schema is a subtype of other, else False.

    is_valid_json method descriptor ¶

    is_valid_json(data: str | bytes) -> bool

    Whether a JSON document parses and is a member of the schema's set.

    Check-only and never raises: malformed JSON, or input that is neither str nor bytes, is simply not a member and returns False. The document is validated in place against the parsed value, with no intermediate Python objects for the structure it walks.

    validate_json method descriptor ¶

    validate_json(
    data: str | bytes, *, fail_fast: bool = False
) -> None

    Validate a JSON document, parsing it on the Rust path.

    Parsing runs in Rust, faster than the standard library, and the parsed value runs the same validation walk as a native object, so this reaches the same decision and the same errors as validate on the parsed object. fail_fast behaves as it does for validate.

    Parameters:

    Name Type Description Default
    data str | bytes

    The JSON document, as str or bytes.

    		 required
    fail_fast bool

    Stop at the first failure instead of aggregating.

    		 False

    Returns:

    Type Description
    None

    None if the document parses and is a member of the schema's set.

    Raises:

    Type Description
    ValidationError

    If the document is malformed JSON (code json_invalid) or is not a member of the schema's set.
    TypeError

    If data is not str or bytes.

    Combinators¶

    union builtin ¶

    union(*schemas: object) -> CompiledValidator

    The union of the given schemas: a value in at least one of their sets.

    intersect builtin ¶

    intersect(*schemas: object) -> CompiledValidator

    The intersection of the given schemas: a value in every one of their sets.

    complement builtin ¶

    complement(schema: object) -> CompiledValidator

    The complement of a schema: every value not in its set.

    ifthen ¶

    ifthen(
    condition: object,
    then: object,
    otherwise: object = anything,
) -> CompiledValidator

    Require then when a value matches condition, else otherwise.

    otherwise admits anything by default. Denotation: (condition and then) or ((not condition) and otherwise); with the default otherwise this is exactly "condition implies then".

    cond ¶

    cond(
    *cases: tuple[object, object],
    default: object = anything,
) -> CompiledValidator

    Select the then of the first matching (condition, then) case.

    A value is checked against each condition in order; the first one it matches selects the then it must satisfy. If it matches no condition it must satisfy default. Equivalent to nesting ifthen from the last case inward, so the earliest matching case wins.

    simplify builtin ¶

    simplify(validator: CompiledValidator) -> CompiledValidator

    An equivalent validator reduced by the lattice laws.

    The result admits exactly the same values in a simpler form (flattened and deduplicated unions and intersections, identities applied, complements in negation-normal form). Returns a new validator; the original is unchanged.

    Structural builders¶

    fixed_sequence builtin ¶

    fixed_sequence(*members: object) -> CompiledValidator

    A fixed-length list matched positionally: element i must satisfy the ith member schema, and the list length must equal the number of members.

    lax builtin ¶

    lax(validator: CompiledValidator) -> CompiledValidator

    Open every record in a schema: undeclared keys are admitted throughout.

    Returns a new validator; the original is unchanged.

    strict builtin ¶

    strict(validator: CompiledValidator) -> CompiledValidator

    Close every record in a schema: only declared keys are admitted throughout.

    The inverse of lax. Returns a new validator; the original is unchanged.

    Recursion¶

    lazy builtin ¶

    lazy(
    builder: Callable[[CompiledValidator], object],
) -> CompiledValidator

    Build a recursive schema as a checked fixpoint.

    builder receives a placeholder validator standing for the schema being defined and returns its body. The placeholder's self-reference is resolved to a back edge, and a non-contractive body — one whose recursive reference is not under a structural constructor — is rejected.
